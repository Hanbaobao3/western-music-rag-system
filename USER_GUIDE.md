# 西方音乐史RAG问答系统 - 用户使用指南

## 📖 系统介绍

西方音乐史RAG问答系统是一个智能的在线问答平台，专门为学习西方音乐史的学生和爱好者设计。系统结合了先进的检索增强生成（RAG）技术和专业音乐知识库，能够准确回答各种音乐史相关问题。

### 🎯 主要特点

- **智能检索**：基于专业的西方音乐史知识库
- **精准回答**：根据问题类型生成格式化的答案
- **知识块化**：按知识点分块，提高检索精度
- **多维度匹配**：关键词、时期、人名等多角度检索
- **用户友好**：简洁明了的界面，易于使用

### 💡 适用人群

- 音乐学学生和研究者
- 西方音乐史爱好者
- 音乐教师和讲师
- 音乐理论学习者
- 对音乐史有疑问的普通用户

## 🚀 快速开始

### 方式一：在线使用（推荐）

1. **访问网址**
   - 打开浏览器，访问：`https://western-music-rag-system.onrender.com`
   - 无需安装任何软件
   - 直接在网页中输入问题

2. **开始问答**
   - 在输入框中输入你的问题
   - 点击"发送"按钮或按Enter键
   - 系统会基于知识库生成回答

### 方式二：本地运行

#### 1. 系统要求

- **操作系统**：Windows、macOS、Linux
- **Python环境**：Python 3.8+ 
- **网络连接**：需要连接互联网（如果使用AI增强功能）
- **存储空间**：至少100MB可用空间

#### 2. 安装步骤

##### Windows用户：

```bash
# 1. 克隆项目
git clone https://github.com/Hanbaobao3/western-music-rag-system.git
cd western-music-rag-system

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动服务器
python app/basic_rag_server.py
```

##### macOS/Linux用户：

```bash
# 1. 克隆项目
git clone https://github.com/Hanbaobao3/western-music-rag-system.git
cd western-music-rag-system

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动服务器
python3 app/basic_rag_server.py
```

#### 3. 访问界面

- 服务器启动后，在浏览器中访问：`http://127.0.0.1:8000`
- 你会看到一个简洁、专业的问答界面

## 📝 使用技巧

### 提问技巧

#### ✅ 高效提问方式

1. **具体明确**
   - 好的提问："贝多芬的《命运交响曲》有什么特点？"
   - 不好的提问："音乐怎么样？"

2. **包含关键信息**
   - 提问时包含人物名、作品名、时期等关键词
   - 例如："海顿的交响曲创作特点"

3. **分步提问**
   - 复杂问题可以拆分成几个简单问题
   - 例如："什么是奏鸣曲式？"然后"奏鸣曲式有哪些部分？"

4. **使用音乐术语**
   - 使用专业术语获得更准确的回答
   - 例如："赋格的对位法特点"、"和声进行的类型"

#### ❌ 避免的提问方式

1. **过于宽泛**
   - 避免："音乐是什么？"、"给我讲讲音乐"
   - 建议聚焦在具体问题

2. **多个问题混合**
   - 一次只问一个问题
   - 避免："贝多芬和莫扎特有什么区别？还有海顿呢？"

3. **缺乏背景**
   - 避免完全离开知识库范围的问题
   - 例如："现代流行音乐的发展史"（知识库主要覆盖古典音乐）

### 功能使用技巧

#### 🎯 时期筛选

- **快速导航**：点击不同时期快速跳转到相关内容
- **针对性学习**：选择某个时期后，可以专注于该时期的音乐
- **历史对比**：通过时期选择了解不同时期的风格演变

#### 📚 历史记录

- **复习重要问题**：查看之前的问答，复习重点内容
- **追踪学习进度**：通过历史记录了解自己的学习进度
- **重新探索**：可以重新询问历史问题，获得更深入的理解

#### 🔧 故障排查

**遇到问题时**：
1. 检查网络连接
2. 查看浏览器控制台错误信息
3. 尝试重新表述问题
4. 清除历史记录重新开始

## 🌟 功能示例

### 示例1：人物询问

**问题**："贝多芬是谁？"

**回答**：
```
贝多芬（Ludwig van Beethoven，1770-1827）是德国作曲家，古典主义音乐的代表人物之一。

主要贡献：
- 确立了交响曲的标准形式
- 创作了32首钢琴奏鸣曲
- 发展了奏鸣曲式的结构
- 在和声进行和配器方面有重要创新

代表作品：《命运交响曲》《田园交响曲》《月光奏鸣曲》等。
```

### 示例2：概念解释

**问题**："什么是赋格？"

**回答**：
```
赋格（Fugue）是一种复调音乐形式，起源于文艺复兴时期，在巴洛克时期达到高峰。

基本特点：
- 声部独立：通常为3-4个独立声部
- 主题重复：一个主题在不同声部中重复出现
- 对位手法：通过模仿、倒影等对位技巧发展

结构：通常分为主题、答题、间插等部分。
```

### 示例3：时期特点

**问题**："巴洛克音乐的主要特点是什么？"

**回答**：
```
巴洛克音乐（Baroque Music，1600-1750）的主要特点包括：

1. 通奏低音的使用
   - 巴洛克音乐的标志性手法
   - 提供和声支持和即兴演奏空间

2. 华丽的装饰性
   - 复杂的装饰音和华丽经过
   - 情感表达强烈

3. 戏剧性对比
   - 强与弱的音量对比
   - 明显的层次感和空间感

4. 对位法的广泛应用
   - 巴赫为代表的对位法大师
   - 多声部的独立线条交织

代表人物：巴赫、亨德尔、维瓦尔第等。
```

### 示例4：作品分析

**问题**："贝多芬的第九交响曲有什么创新？"

**回答**：
```
贝多芬的第九交响曲（《合唱》）在其创作中具有重要意义：

创新点：
1. 引入合唱：首次在交响曲中加入人声合唱
2. 巨大的规模：超越了传统交响曲的结构限制
3. 革命性的第四乐章：从传统的快板变为庄严的合唱曲
4. 技术难度：对乐团的技术要求达到空前高度

影响：
- 成为后世交响曲创作的重要参考
- 展现了贝多芬晚期的艺术理想
- 体现了个人对社会进步的期望
```

## 🔧 故障排查

### 常见问题及解决方案

#### 问题1：无法启动服务器

**症状**：启动时出现端口占用错误

**解决方案**：
```bash
# 查找占用端口的进程
netstat -ano | grep ":8000"

# 停止占用进程
taskkill /F /PID 进程ID

# 或使用不同端口
python app/basic_rag_server.py
```

#### 问题2：网页显示错误信息

**症状**：界面显示错误消息或无法加载

**解决方案**：
1. **检查网络连接**：确保网络正常
2. **清除浏览器缓存**：按Ctrl+F5强制刷新页面
3. **查看控制台错误**：按F12查看详细错误信息
4. **重试操作**：刷新页面或重启服务器

#### 问题3：答案不准确或不完整

**症状**：系统回答与知识库不符

**解决方案**：
1. **优化提问方式**：使用更具体和准确的表述
2. **检查知识库**：确认相关信息存在于知识库中
3. **分步提问**：将复杂问题拆分为简单问题
4. **使用专业术语**：使用正确的音乐术语

#### 问题4：响应速度慢

**症状**：提交问题后长时间没有响应

**解决方案**：
1. **检查网络连接**：确保网络稳定
2. **减少知识库规模**：删除不必要的内容
3. **重启服务器**：清除内存缓存
4. **使用本地版本**：避免网络延迟

### 获取帮助

如果遇到上述常见问题无法解决：

1. **查看日志**：检查服务器启动和运行日志
2. **浏览器调试**：使用开发者工具查看详细错误
3. **重新部署**：在Render Dashboard中重新部署
4. **联系我们**：通过GitHub Issues报告问题

## 🔐 系统技术说明

### RAG技术架构

本系统采用检索增强生成（RAG）技术：

- **检索阶段**：从知识库中检索最相关的内容
- **增强阶段**：基于检索内容生成准确的回答
- **生成阶段**：确保回答的准确性和相关性

### 知识库结构

- **文本切块**：按知识点智能分块，提高检索精度
- **多维匹配**：关键词、时期、人名等多角度匹配
- **重要性评分**：对不同内容进行重要性评估

### 数据处理

- **UTF-8编码**：确保多语言内容正确显示
- **错误处理**：优雅处理各种异常情况
- **响应优化**：快速响应，良好的用户体验
